## 바이브코딩이란?

**바이브코딩(Vibe Coding)** = AI에게 "이렇게 해줘"라고 말하면 코드를 생성해주는 개발 방식

### 기존 개발 vs 바이브코딩

| 기존 개발 | 바이브코딩 |
|-----------|------------|
| 코드를 직접 타이핑 | AI에게 자연어로 요청 |
| 문법을 외워야 함 | 원하는 것만 설명하면 됨 |
| 에러를 직접 해결 | AI가 에러 분석 및 수정 |

### 핵심 원칙

1. **구체적으로 요청하기** - "버튼 만들어줘" (X) → "파란색 로그인 버튼을 화면 중앙에 만들어줘" (O)
2. **결과 확인하기** - AI가 만든 코드가 의도대로 동작하는지 항상 확인
3. **조금씩 요청하기** - 한 번에 많은 기능보다 하나씩 요청

## 안티그래비티 기본 사용법

### 핵심 단축키

| 단축키 | 기능 |
|--------|------|
| `Ctrl + L` | AI 채팅창 열기 (대화하며 코딩) |
| `Ctrl + K` | 선택한 코드에 대해 AI에게 질문 |
| `` Ctrl + ` `` | 터미널 열기/닫기 |
| `Ctrl + S` | 파일 저장 |
| `Ctrl + Shift + P` | 명령어 팔레트 (모든 기능 검색) |

### AI에게 요청하는 방법

**방법 1: 채팅으로 요청 (Ctrl + L)**
```
"로그인 페이지를 만들어줘. 이메일과 비밀번호 입력 필드가 있고,
로그인 버튼을 누르면 /api/auth/login으로 요청을 보내줘"
```

**방법 2: 코드 선택 후 질문 (Ctrl + L)**
- 이해 안 되는 코드 선택 → `Ctrl + L` → "이 코드가 뭐하는 건지 설명해줘"
- 에러난 코드 선택 → `Ctrl + L` → "이 에러 왜 나는지 고쳐줘"

## .antigravityrules 파일 이해하기

### 이게 뭔가요?

`.antigravityrules`는 **AI에게 주는 지침서**입니다.

- 프로젝트 루트에 위치
- AI가 코드를 생성할 때 이 규칙을 따름
- 일관된 코드 스타일 유지에 도움

### 핵심 내용

```
# .antigravityrules 예시

## 프로젝트 구조
- 백엔드: server/ 폴더
- 프론트엔드: client/ 폴더

## 코딩 규칙
- Python: 함수명은 snake_case (예: get_user_info)
- TypeScript: 변수명은 camelCase (예: getUserInfo)
- 한 함수는 하나의 기능만

## 기술 스택
- 백엔드: FastAPI + SQLAlchemy
- 프론트엔드: React + Tailwind CSS
```

### 어떻게 활용하나요?

**AI에게 요청할 때:**
- AI가 자동으로 `.antigravityrules`를 읽고 규칙에 맞게 코드 생성
- 별도로 "이 규칙 따라줘"라고 말할 필요 없음

**규칙 추가하고 싶을 때:**
```
"앞으로 버튼은 항상 파란색으로 만들어줘"
→ .antigravityrules에 추가 요청
```

## Alembic(알렘빅) - DB 변경 관리

### 알렘빅이 뭔가요?

데이터베이스 테이블 구조를 변경할 때 사용하는 도구입니다.

**비유하자면:**
- 엑셀 시트의 열(컬럼)을 추가/삭제/수정하는 것
- 그 변경 이력을 기록해두는 것

### 왜 필요한가요?

```
문제 상황:
1. 처음에 "사용자" 테이블에 이름, 이메일만 있었음
2. 나중에 "전화번호"도 저장해야 함
3. 기존 데이터는 유지하면서 전화번호 열만 추가해야 함

→ 알렘빅이 이걸 안전하게 처리해줌!
```

### 사용법 (딱 2가지만 기억!)

**1. 변경 감지하고 기록 만들기**
```bash
alembic revision --autogenerate -m "전화번호 필드 추가"
```
- `models/` 폴더의 변경사항을 자동 감지
- `migration/versions/` 폴더에 변경 기록 파일 생성

**2. 변경 적용하기**
```bash
alembic upgrade head
```
- 실제 데이터베이스에 변경 적용

### 주의사항

- **코드(models/)를 먼저 수정** → 그 다음 알렘빅 실행
- 변경 기록 파일 생성 후 **꼭 내용 확인**하고 적용

## 웹페이지 디자인 가이드

### 디자인 아이디어 얻는 법

#### 1. 마음에 드는 페이지 캡처해서 활용

```
AI에게 요청:
"[첨부한 스크린샷]처럼 디자인해줘"
또는
"이 이미지와 비슷한 스타일로 로그인 페이지 만들어줘"
```

**캡처 방법:**
- Windows: `Win + Shift + S`
- Mac: `Cmd + Shift + 4`

#### 2. 트렌드 키워드로 요청

```
"2026년 트렌드 스타일로 대시보드 만들어줘"
"미니멀한 핀테크 웹사이트 스타일로 만들어줘"
"애플 공식 홈페이지 느낌으로 만들어줘"
```

#### 3. 구체적인 스타일 지정

```
"깔끔한 화이트 배경에 파란색 포인트 컬러로"
"카드 형태로 정보를 보여주는 레이아웃으로"
"왼쪽에 사이드바, 오른쪽에 메인 콘텐츠 구조로"
```

### 자주 쓰는 디자인 요청 예시

| 원하는 것 | 요청 예시 |
|-----------|-----------|
| 깔끔한 로그인 | "미니멀한 로그인 페이지, 중앙 정렬, 그림자 효과" |
| 대시보드 | "좌측 사이드바, 상단에 통계 카드 4개, 아래에 그래프" |
| 테이블 | "깔끔한 데이터 테이블, 페이지네이션, 검색 기능" |
| 폼 입력 | "세련된 입력 폼, 유효성 검사 표시" |

### 참고하면 좋은 디자인 사이트

- **Dribbble** (dribbble.com) - 디자인 영감
- **Tailwind UI** (tailwindui.com) - 컴포넌트 예시
- **shadcn/ui** (ui.shadcn.com) - 무료 컴포넌트

## 자주 쓰는 명령어 모음

### 터미널 기본 명령어

```bash
# 현재 위치 확인
pwd

# 폴더 이동
cd client          # client 폴더로 이동
cd ..              # 상위 폴더로 이동
cd ~               # 홈 폴더로 이동

# 폴더 내용 보기
ls                 # Mac/Linux
dir                # Windows
```

### 백엔드 명령어

```bash
# 가상환경 활성화
source .venv/bin/activate  # Mac/Linux
.venv\Scripts\activate     # Windows

# 서버 실행
python -m server.main

# 패키지 설치
pip install 패키지이름

# DB 마이그레이션
alembic revision --autogenerate -m "설명"
alembic upgrade head
```

### 프론트엔드 명령어

```bash
# client 폴더에서 실행
cd client

# 의존성 설치
npm install

# 개발 서버 실행
npm run dev

# 특정 패키지 설치
npm install 패키지이름
```

### Git 명령어

```bash
# 변경사항 확인
git status

# 모든 변경사항 저장 준비
git add .

# 저장 (커밋)
git commit -m "변경 내용 설명"

# 서버에 업로드
git push

# 서버에서 최신 코드 받기
git pull
```

## 초보자 필수 체크리스트

### 개발 시작 전

- [ ] 안티그래비티 실행됨
- [ ] 터미널에서 프로젝트 폴더에 있는지 확인 (`pwd`)
- [ ] 백엔드 가상환경 활성화 확인 (프롬프트에 `(.venv)` 표시)
- [ ] 백엔드 서버 실행 중 (`python -m server.main`)
- [ ] 프론트엔드 서버 실행 중 (`npm run dev`)

### 코드 수정 후

- [ ] 파일 저장했는지 확인 (`Ctrl + S`)
- [ ] 브라우저에서 결과 확인
- [ ] 에러 메시지 없는지 터미널 확인

### 하루 끝날 때

- [ ] 변경사항 Git에 저장
  ```bash
  git add .
  git commit -m "오늘 작업 내용"
  git push
  ```

## 에러 대처법

### "그냥 AI에게 물어보세요!"

에러가 나면:
1. 에러 메시지 전체 복사
2. AI 채팅창에 붙여넣기
3. "이 에러 왜 나는 거야? 어떻게 고쳐?" 라고 질문

### 자주 나는 에러와 해결법

| 에러 | 원인 | 해결 |
|------|------|------|
| `ModuleNotFoundError` | 패키지 미설치 | `pip install 패키지명` |
| `command not found` | 가상환경 미활성화 | `source .venv/bin/activate` |
| `Connection refused` | 서버 안 켜짐 | 백엔드/프론트엔드 서버 실행 |
| `404 Not Found` | URL 경로 오류 | API 경로 확인 |
| `500 Internal Server Error` | 백엔드 에러 | 터미널에서 에러 메시지 확인 |

### 최후의 수단

1. **터미널 전부 끄고 다시 시작**
2. **안티그래비티 재시작**
3. **컴퓨터 재시작**
4. **팀원에게 도움 요청**

## 용어 정리

| 용어 | 쉬운 설명 |
|------|-----------|
| **프론트엔드** | 사용자가 보는 화면 (버튼, 입력창 등) |
| **백엔드** | 서버에서 돌아가는 로직 (데이터 처리, 저장) |
| **API** | 프론트엔드와 백엔드가 대화하는 방법 |
| **데이터베이스 (DB)** | 데이터를 저장하는 곳 (엑셀 파일 같은 것) |
| **컴포넌트** | 재사용 가능한 UI 조각 (레고 블록 같은 것) |
| **라우터** | URL 주소에 따라 다른 페이지를 보여주는 것 |
| **가상환경** | 프로젝트별로 분리된 Python 환경 |
| **마이그레이션** | DB 구조 변경을 기록하고 적용하는 것 |
| **Git** | 코드 버전 관리 도구 (저장 이력 관리) |
| **커밋(Commit)** | Git에 변경사항을 저장하는 것 |
| **푸시(Push)** | 내 컴퓨터의 커밋을 서버에 업로드 |
| **풀(Pull)** | 서버의 최신 코드를 내 컴퓨터로 다운로드 |
